TIDES

A Mutable Instruments modulation source for Ableton Move — the first of its kind.

Complex function generator → MIDI CC output → modulate anything in the chain. Filter sweeps, rhythmic gates, evolving textures — all from one module.

01 — The Concept

What if the Move had a real modulation matrix?

Ableton Move's synth engines sound great but lack deep modulation. No complex LFOs. No function generators. No way to create evolving, breathing patches that change over time.

Tides changes that. It's a port of Mutable Instruments' Tides v2 — widely regarded as one of the most versatile modulation sources in Eurorack — reimagined as a MIDI FX module for Move Everything. Instead of outputting control voltage, it outputs MIDI CC to modulate any parameter on the synth loaded in the chain.

◇

3 Modes

AD envelope (one-shot), free-running LFO, and AR envelope (gate-held). From percussive plucks to infinite sweeps.

◈

4 Outputs

Each generating a related but different modulation signal. Route each to a different synth parameter for complex, interconnected movement.

◆

Infinite Shapes

Continuously morph between triangle, saw, square, and beyond. Add wavefolding. Smooth or sharpen. Shapes that basic LFOs can't touch.

◉

Smart Routing

No CC numbers. Just scroll through parameter names: "Filter Cutoff," "FM Amount," "Osc Mix." The module knows what synth is loaded.

02 — Signal Flow

Where it lives in the chain

Tides sits in the MIDI FX slot — the same position as the SuperArp arpeggiator. It receives note data from the pads and outputs MIDI CC to the synth engine downstream.

Pads / Sequencer → TIDES MOD SOURCE → CC Synth (DX7 / Surge / etc.) → Audio FX → 🔊

Note-on triggers envelopes · Velocity modulates depth · Transport syncs LFO rate
Up to 4 CC outputs → 4 different param destinations simultaneously
03 — The Killer Feature

Destinations by name, not number

The module ships with parameter maps for every Move Everything synth. When a synth is loaded, Tides reads its manifest and shows you the available destinations as readable names — not raw MIDI CC numbers.

05 — Why This Matters

First of its kind

1

No modulation sources exist in Move Everything

The ecosystem has synths, audio FX, and one arpeggiator. There are zero modules that generate continuous modulation and route it to other modules' parameters. This opens a new category.

2

The param manifest system benefits everyone

Once synths expose their parameters via manifests, any future module can use them — macro controllers, random modulators, step sequencers targeting specific params. We'd be laying infrastructure.

3

Tides is the right DSP for the job

MIT-licensed, well-documented, battle-tested in thousands of Eurorack systems. The v2 codebase shares architectural patterns with Plaits (already ported to Move). It's portable and proven.

4

Content opportunity

First modulation source for Move Everything is inherently newsworthy in the community. Tutorial/demo potential on YouTube. Positions us as contributors, not just users.

06 — Roadmap

Three phases

Core MVP

  • Port Tides v2 DSP (strip hardware deps)
  • MIDI FX plugin skeleton
  • Looping mode (free-run LFO)
  • Single CC output, fixed destination
  • Basic encoder mapping
  • Minimal OLED display
  • Build + test on Move hardware

Full Feature Set

  • AD and AR ramp modes
  • All 4 output modes
  • 4-destination routing
  • Param manifest + name picker
  • Mod depth per output
  • Tempo sync to transport
  • Waveform preview on OLED
  • Bipolar / unipolar toggle

Ecosystem

  • Bundled maps for all ME synths
  • Propose discovery API to community
  • Preset save/load
  • Community testing + Discord feedback
  • Module Store submission
  • YouTube demo / walkthrough
07 — Open Questions

Things to figure out together

Does Move Everything's MIDI FX slot pass CC downstream to synth modules? The SuperArp passes notes, but CC passthrough is unconfirmed. This is the #1 technical validation needed.

CC bandwidth — how many CC messages per second can the internal bus handle before things get choppy? This determines our output rate (30Hz? 60Hz? 120Hz?).

Can one MIDI FX module modulate both the synth AND an audio FX? E.g. Tides output 1 → filter cutoff on DX7, output 2 → delay feedback on Space Delay.

What CCs do each Move Everything synth actually respond to? We need to audit every synth module's source code to build the param manifests.

Is the Move Everything community (Charles Vestal) open to framework-level additions like a param discovery API?
